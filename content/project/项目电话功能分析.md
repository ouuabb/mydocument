# 项目电话功能实现分析

## 1. 概述

本项目中实现了多种与电话相关的功能，包括拨打电话、获取用户手机号、地址联系电话等。本文档详细分析项目中电话功能的实现方式和使用场景。

## 2. 核心API

### 2.1 拨打电话 API

**`uni.makePhoneCall`** - 打开手机拨号界面并填充号码

```javascript
uni.makePhoneCall({
    phoneNumber: '电话号码',
    success: (res) => {
        console.log('调用成功');
    },
    fail: (err) => {
        console.log('调用失败', err);
    }
});
```

### 2.2 获取用户信息 API

**`ApiGet_getMyInfo()`** - 获取当前用户的个人信息（含手机号）

```javascript
async getUserInfo() {
    try {
        const res = await ApiGet_getMyInfo()
        this.reservePhone = res.user.phonenumber || ''
    } catch (error) {
        console.error('请求获取用户信息失败', error)
    }
}
```

### 2.3 获取地址列表 API

**`getSecondHandGoodAddressList()`** - 获取用户地址列表（含联系电话）

```javascript
async getAddressList() {
    const res = await getSecondHandGoodAddressList({
        addressBo: { page: 0, pageSize: 0, defaultStatus: true }
    })
    this.addressList = res.data.records || []
}
```

## 4. 功能分析

### 4.1 拨打电话流程

1. **用户触发**：点击联系按钮
2. **显示确认**：部分场景会先显示确认弹窗
3. **调用API**：调用 `uni.makePhoneCall`
4. **系统处理**：自动打开手机拨号界面
5. **用户操作**：用户点击拨号键完成拨打

### 4.2 电话获取方式

| 场景 | 电话来源 | 获取方式 |
|------|---------|----------|
| 自提订单 | 预留电话 | 用户个人信息 + 手动输入 |
| 快递订单 | 收货电话 | 地址列表选择 |
| 联系商家 | 商家电话 | 订单数据中获取 |
| 联系服务者 | 服务者电话 | 订单数据中获取 |

## 5. 代码优化建议

### 5.1 统一电话处理工具函数

```javascript
// utils/phoneUtils.js
/**
 * 拨打电话
 * @param {string} phone - 电话号码
 * @param {boolean} showConfirm - 是否显示确认弹窗
 */
export function callPhone(phone, showConfirm = true) {
    if (!phone) {
        uni.showToast({ title: '暂无联系电话', icon: 'none' });
        return;
    }

    const doCall = () => {
        uni.makePhoneCall({
            phoneNumber: phone,
            fail: () => {
                uni.showToast({
                    title: '拨打失败，可手动拨打',
                    icon: 'none',
                    duration: 2000
                });
            }
        });
    };

    if (showConfirm) {
        uni.showModal({
            title: '拨打电话',
            content: `确定拨打 ${phone} 吗？`,
            success: (res) => {
                if (res.confirm) {
                    doCall();
                }
            }
        });
    } else {
        doCall();
    }
}
```

### 5.2 手机号验证

```javascript
// 验证手机号格式
export function validatePhone(phone) {
    return /^1[3-9]\d{9}$/.test(phone);
}

// 使用示例
if (!validatePhone(this.reservePhone)) {
    uni.showToast({ title: '请输入正确的手机号', icon: 'none' });
    return;
}
```

### 5.3 电话数据管理

建议将电话相关的数据管理集中化，避免重复代码：

```javascript
// 统一获取用户手机号
export async function getCurrentUserPhone() {

    }
}
```

## 6. 总结

本项目通过 `uni.makePhoneCall` API 实现了多种场景下的拨打电话功能，配合用户信息和地址管理，为用户提供了便捷的联系方式。主要特点：

- **多场景覆盖**：订单、客服、服务者等多种场景
- **用户友好**：确认弹窗、错误处理等
- **数据来源**：用户信息、地址列表、订单数据
- **跨平台兼容**：uni-app 跨平台实现

通过统一的工具函数和验证机制，可以进一步提高代码的可维护性和用户体验。
