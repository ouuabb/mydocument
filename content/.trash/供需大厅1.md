# 供需管理参数传递规则


## 接口信息

  

- **接口地址**：`/api/goods/supplyDemandInfo/myList`

- **请求方式**：POST

  

## 参数说明

  

### 基本参数

| 参数名          | 类型            | 说明    | 取值范围                                             |
| ------------ | ------------- | ----- | ------------------------------------------------ |
| page         | Number        | 当前页码  | 1, 2, 3, ...                                     |
| pageSize     | Number        | 每页记录数 | 10, 20, 50, ...                                  |
| searchFlag   | Number        | 查询类型  | 1: 我的发布, 2: 我的响应                                 |
| infoType     | Number        | 信息类型  | 1: 供应大厅, 2: 需求大厅                                 |
| statusSearch | String/Number | 状态筛选  | 空字符串: 全部, 1: 待响应, 2: 待选择, 3: 待服务, 4: 服务中, 5: 已完成 |



### 状态含义

  

| statusSearch | 状态名称 | 适用场景 |
|-------------|----------|----------|
| 1 | 待响应 | 需求大厅-我的发布 |
| 2 | 待选择 | 需求大厅-我的发布<br>需求大厅-我的响应<br>供应大厅-我的响应 |
| 3 | 待服务 | 需求大厅-我的发布<br>需求大厅-我的响应<br>供应大厅-我的发布<br>供应大厅-我的响应 |
| 4 | 服务中 | 需求大厅-我的发布<br>需求大厅-我的响应<br>供应大厅-我的发布<br>供应大厅-我的响应 |
| 5 | 已完成 | 需求大厅-我的发布<br>需求大厅-我的响应<br>供应大厅-我的发布<br>供应大厅-我的响应 |

## 前端实现逻辑

  

### 状态切换

  

1. **需求/供应大厅切换**：修改 `infoType` 值

   - 需求大厅: `infoType = 2`

   - 供应大厅: `infoType = 1`

  

2. **我的发布/我的响应切换**：修改 `searchFlag` 值

   - 我的发布: `searchFlag = 1`

   - 我的响应: `searchFlag = 2`

  

3. **状态标签切换**：修改 `current` 值，对应获取 `list2` 或 `list3` 中的 `statusSearch` 值

   - 需求大厅-我的发布: 使用 `list2`

   - 其他场景: 使用 `list3`

  

### 示例代码

  

```javascript

// 需求大厅-我的发布-待服务

const params = {

  page: 1,

  pageSize: 10,

  searchFlag: 1,  // 我的发布

  infoType: 2,    // 需求大厅

  statusSearch: 3 // 待服务

};

  

// 供应大厅-我的响应-全部

const params = {

  page: 1,

  pageSize: 10,

  searchFlag: 2,  // 我的响应

  infoType: 1,    // 供应大厅

  statusSearch: "" // 全部

};

```

  

## 后端处理建议

  

后端接口应根据 `statusSearch` 参数值过滤返回数据，确保只返回与当前选中状态匹配的记录。如果 `statusSearch` 为空字符串，则返回所有状态的记录。
